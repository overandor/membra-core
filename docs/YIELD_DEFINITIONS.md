# MEMBRA Yield Definitions

## What MEMBRA Means by "Yield"

MEMBRA uses "yield" to mean **validated productive output**.

MEMBRA does **not** guarantee financial income.

## Four Kinds of Yield

### 1. Artifact Yield

Files, code, documentation, datasets, screenshots, reports, or any tangible output produced by a job.

- Measured by: number of files, size, diversity, completeness
- Generated by: MEMBRA directly
- Example: "7 files created, 18.4 KB total, including app.py, README.md, tests/"

### 2. Validation Yield

Tests passed, benchmarks completed, audits cleared, policy checks satisfied, lint clean, security scan passed.

- Measured by: test pass rate, benchmark scores, security flags count
- Generated by: MEMBRA directly
- Example: "18/18 tests passed, 0 security flags, lint clean"

### 3. Market Yield

Bounty accepted, invoice paid, grant approved, customer purchased, or any third-party economic recognition of the output.

- Measured by: confirmed receipts from external systems
- **NOT generated by MEMBRA directly**
- Documented only when external receipts exist
- Example: "Bounty paid: $500, Invoice #1234 confirmed"

### 4. Chain Yield

Blockchain or protocol receipts: devnet/mainnet transaction, escrow release, staking reward, liquidity provision receipt.

- Measured by: on-chain transaction confirmations
- **NOT generated by MEMBRA directly**
- Documented only when external receipts exist
- Example: "Solana devnet tx: 5fae... confirmed in slot 234567890"

## What MEMBRA Can and Cannot Do

| Capability | Status |
|------------|--------|
| Generate artifacts (code, docs, tests) | ✅ Yes |
| Run tests and benchmarks | ✅ Yes |
| Score quality and completeness | ✅ Yes |
| Validate with multiple validators | ✅ Yes |
| Create proof bundles with hashes | ✅ Yes |
| Guarantee someone will buy the output | ❌ No |
| Generate real revenue without external demand | ❌ No |
| Create on-chain value without external protocols | ❌ No |
| Promise financial returns to users | ❌ No |

## The Honest Formula

```
Total Score =
  (Artifact Yield × 0.35)
+ (Validation Yield × 0.35)
+ (Market Yield × 0.15)
+ (Chain Yield × 0.15)
```

MEMBRA controls the first two.
MEMBRA documents the last two only when receipts exist.

## Legal Protection

This document exists to protect MEMBRA from misrepresentation:

- MEMBRA is a **job accounting and validation system**, not an investment platform.
- MEMBRA **measures output quality**, it does not guarantee financial returns.
- MEMBRA **creates proof bundles**, it does not create market demand.
- Any revenue or yield from MEMBRA outputs depends on **external buyers, bounties, grants, or customers**.
- MEMBRA users are responsible for their own **compliance, security, and financial decisions**.

## For Investors

MEMBRA turns chat into executable economic units.

A prompt becomes a containerized job.
The job produces artifacts.
Artifacts are scored as yield.
Validators reach consensus.
The resulting proof bundle can be sold, audited, anchored, or settled.

MEMBRA **enables** economic activity by making AI work auditable.
MEMBRA does not **create** economic activity by magic.

## For Users

"Don't just chat. Ship provable work."

MEMBRA gives you:
1. A structured way to turn prompts into jobs
2. Automated validation of your output
3. A proof bundle you can show to customers, grantors, or bounty platforms
4. A yield score that measures quality, not hype

MEMBRA does not give you:
1. Guaranteed income
2. Automatic buyers
3. Risk-free deployment
4. Financial advice

## Summary

> **Artifact yield is produced when a job creates useful files.**
> **Validation yield is produced when tests, checks, or benchmarks pass.**
> **Market yield exists only when a third party pays, accepts, funds, or purchases the output.**
> **Chain yield exists only when an external blockchain or protocol receipt proves settlement.**
>
> **MEMBRA can measure and package proof. It cannot manufacture external demand.**
